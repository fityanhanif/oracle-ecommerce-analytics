## Interview talking points

### Why Oracle?

I wanted to show database development skills beyond notebook analysis. Oracle forced me to think about schema design, load validation, data types, constraints, query views, and PL/SQL routines.

### What was the strongest insight?

Late delivery has the clearest relationship with customer dissatisfaction. Late orders had 2.57 average review score and 54.02% low-review rate. Orders that were not late had 4.21 average review score and 11.38% low-review rate.

### What would I improve next?

I would add a clean warehouse layer with fact and dimension tables, then connect the dashboard to a scheduled export or a lightweight API. I would also add Portuguese NLP analysis for review text if the goal expands beyond Oracle operations analytics.

### What skills does it show?

Data engineering, Oracle SQL, PL/SQL, database modeling, data quality validation, business analysis, BI dashboarding, and documentation.

### How does it match database-support roles?

It shows strong SQL through Oracle analytics views and reporting queries, Oracle database development through schema objects and PL/SQL, and PostgreSQL basics through monitoring query templates for table growth, active sessions, blocking sessions, slow queries, index usage, data freshness, and reporting cadence.
